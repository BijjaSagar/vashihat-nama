1. Project Overview

What is Vasihat Nama and why does it matter?

Vasihat Nama (Urdu: "Letter of Advice") is a cross-platform mobile application that solves a critical real-world problem: What happens to your digital assets when you're no longer around?

The app provides a secure, encrypted vault where users store sensitive information — bank accounts, passwords, legal documents, credit cards — and designate nominees who automatically receive access when the user fails to perform regular biometric "Proof of Life" check-ins.

The Problem We Solve

💰

Digital Asset Loss

Billions in crypto, bank accounts & digital assets are lost annually because heirs lack access credentials.

📄

Document Scattering

Important legal documents (wills, deeds, policies) are scattered across physical and digital locations.

❤️

Life Verification

No automated way to verify a person is alive and trigger asset transfer upon death or incapacitation.

🌍

Regional Compliance

Different countries have different legal requirements for estate planning and document management.

2. Core Philosophy & Key Points

The five pillars that define Vasihat Nama

🔐 Zero-Knowledge Security

The server never sees plaintext data. All vault items are encrypted client-side using RSA/AES before transmission. Even the server operator cannot read user data.

❤️ Proof of Life (Dead Man's Switch)

Users set a check-in frequency (e.g., every 30 days). Biometric authentication (fingerprint/face) is required for each check-in. If the user fails to check in, nominees are automatically notified and granted access.

🧑‍🤝‍🧑 Selective Nominee Access

Each vault item can be individually assigned to specific nominees. Supports both Digital (email/app) and Physical (hand-delivery with identity verification) delivery modes.

🤖 AI-Powered Intelligence

AI Will Drafter with voice input, Smart OCR document scanning, credit card detection, document classification, conflict checking, tone analysis, and an AI Legal Assistant chatbot.

📊 Security Health Score

Gamified 0-100% score with a visual checklist. One-tap "Fix" buttons navigate directly to resolve each security gap.

8. Security Architecture

Zero-knowledge encryption model

Encryption Flow

1. User creates vault item (e.g., password) │ 2. Generate random AES-256 key │ 3. Encrypt data with AES key ──▸ Encrypted Blob │ 4. Encrypt AES key with User's RSA Public Key ──▸ Encrypted Key │ 5. Send BOTH encrypted blobs to server ⚠️ Server NEVER sees plaintext ═══════════════════════════════════════════════════ NOMINEE ACCESS GRANT: 6. Decrypt AES key with User's Private Key 7. Re-encrypt AES key with Nominee's RSA Public Key 8. Store nominee's encrypted key separately 9. Nominee decrypts with THEIR Private Key (without ever needing User's Private Key)

Authentication Security

Layer Mechanism
Login OTP via SMS — 6-digit code with 5-minute expiry
Check-In Device biometrics — Fingerprint / Face Recognition
No Passwords Authentication is mobile number + OTP only
Audit Trail All OTP attempts and check-ins are logged
